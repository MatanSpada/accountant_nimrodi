# Design Review — Current State

## What is wrong

### Hero section on every page

Every page opens with a `.hero` two-column banner containing a headline and a small panel with a "tip" or label. This pattern was likely generated by an AI to fill space. It adds 3–4 lines of text that nobody reads before they can see the actual content. On the payment details page, the hero says the customer name — then the same name appears again in the summary card below. Redundant and noisy.

### Dashboard is not an executive overview

The current dashboard is structurally: hero → metrics card → split with table + breakdown + quick actions. The "metrics card" is inside a white card with its own section header. The KPIs feel buried and secondary. A Power BI dashboard leads with numbers at the very top, not nested inside a card-within-a-card layout.

### "Section header description" copy everywhere

Every `.section-head` has a `<p>` with an explanatory sentence. On the new payment page: "המידע כאן משמש ליצירת הקישור ולתצוגה ברשימת העסקאות." This is filler. It reads as AI padding and adds visual noise without communicating anything a professional user needs.

### Sidebar has too much copy

The sidebar brand block contains a 2-line paragraph explaining what the system does. A professional tool does not narrate itself. The sidebar footnote ("ממשק ניהול שקט, מדויק וברור לעבודה שוטפת מול לקוחות...") adds no value.

### Navigation labels are imprecise

- "לוח בקרה" is fine for dashboard
- "יצירת בקשה" is action-in-nav, which breaks the pattern (other items are nouns)
- "עסקאות" is clean

Nav links have a border and background even in inactive state, which makes every link look "selected."

### Warm beige is soft but not premium

The `--bg: #f4f2ee` and `--surface: #fffdf8` palette is pleasant but reads as a lifestyle brand, not a finance back-office tool. Power BI uses cool neutral grays with high-contrast accent. The current palette is not wrong, but needs to shift slightly cooler and more confident.

### Metric cards are a 5-column grid

On a wide screen this is fine. But the metrics have no visual weight hierarchy — all five cards look identical. In a premium BI dashboard, the primary KPIs (total paid, total amount) get visual prominence; secondary counts are smaller.

### The table on dashboard duplicates the payments list

The dashboard shows an 8-column table identical to the payments list. This is cognitively heavy for an overview page. The dashboard should show a simplified recent-activity view, not a full table.

### Forms have no visual grouping clarity

The create payment form uses a `.form-grid.two-columns` that works, but the form sits inside a full-width card without a centered max-width constraint. Wide forms (1200px+) look stretched and hard to fill.

### Payment details shows technical info too prominently

The technical section is inside a `<details>` — this is good. But the page starts with a hero showing the customer name, then repeats it in the summary grid. The status badge is in the hero panel (secondary position) when it should be the most prominent thing on the page.

### Context chips in the topbar feel like debug output

The topbar shows "סביבה: פיתוח" and "מצב תשלום: דמו" as pill chips. This is developer-facing info that clutters the header of every page for all users.

### No visual differentiation between page types

Dashboard, form, list, and details all use the same layout shell. A premium product gives the dashboard a slightly different feel (more open, more breathing room) compared to operational task pages (more compact, focused).
